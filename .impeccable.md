# SpiceHub Design Context

## Users
Home cooks and meal planners who collect recipes from social media (Instagram, TikTok, blogs). They want to quickly import recipes while scrolling, on mobile, without friction. They value reliability and transparency—they want to know if a recipe was fully extracted or needs manual review.

## Use Cases
- Share an Instagram post directly to SpiceHub → recipe imports automatically
- Paste a blog URL → extract ingredients and directions
- Copy a TikTok caption → AI structures it into a recipe
- Fallback: manually refine caption when extraction fails (with clear explanation of why)

## Brand Personality
**3 words**: Warm, Approachable, Reliable

Inspired by Paprika 3 (competent recipe manager) but with playful energy ("Meal Spinner" concept). The brand should feel like a trusted friend who makes cooking fun and effortless.

## Aesthetic Direction
- **Theme**: Light, warm (off-white background #fff8f0 with orange primary #e65100)
- **Tone**: Playful but competent—show personality without sacrificing trust
- **Reference feel**: Inviting kitchen reference book, reliable but with character
- **Anti-reference**: Clinical, corporate, overly minimalist (feels impersonal)

## Design Principles for This Project

1. **Transparency over Magic**: Show the import pipeline. Users should understand why a recipe might need manual review. Explain each phase.

2. **Reassurance & Feedback**: Clear progress indicators, explicit next steps, reassuring copy that acknowledges the complexity behind the scenes.

3. **Integrated, Not Separate**: The BrowserAssist stepper should feel like a natural part of the import modal flow, not a detached "advanced" system.

4. **Playful Without Frivolous**: Micro-interactions and personality are welcome—but only when they reinforce trust or delight. No decorative elements.

5. **Mobile-First Touch UX**: Large touch targets, clear drop zones, swipe-friendly interactions. Every interactive surface should feel confident and responsive.

6. **Offline-First Mindset**: Always assume the connection might drop. Show offline queueing, retry logic, and graceful fallbacks.

## Technical Constraints
- React 19 + Vite, IndexedDB (Dexie), offline-first PWA
- No backend—all processing client-side (yt-dlp, Instagram embeds, Puppeteer agent, Gemini AI)
- Mobile-optimized (max 600px, touch-friendly)

## Current Design System
- **Colors**: Primary orange #e65100, Accent green #2e7d32, Warm bg #fff8f0
- **Typography**: System fonts (Segoe UI/SF Pro) + Press Start 2P for playful accents
- **Spacing**: 4pt scale (4, 8, 16, 24, 32, 48)
- **Radius**: 16px (default), 10px (sm), 20px (lg)
- **Motion**: Spring easing, 200-300ms transitions
